Veille Technologique

Sujet : La virtualisation et le cloud computing dans les infrastructures modernes

J'ai choisi de faire ma veille sur la virtualisation et le cloud parce que c'est ce que je pratique le plus dans mes projets — VMware, KVM, OpenStack. C'est aussi ce qui recrute en ce moment dans les métiers SISR, donc autant rester à jour sur les évolutions.
// concepts.clés

Concepts clés

🖥️
Virtualisation
En gros : faire tourner plusieurs OS sur une seule machine physique. J'ai pratiqué avec VMware ESXi et KVM — les hyperviseurs de type 1 tournent directement sur le métal, sans OS hôte en dessous. C'est ce qu'on retrouve dans tous les data centers aujourd'hui.
☁️
Cloud Computing
Des ressources (serveurs, stockage, réseau) accessibles à la demande sur Internet. IaaS, PaaS, SaaS — trois façons différentes de consommer le cloud. AWS, Azure et GCP se partagent le marché. Les offres hybrides (un pied on-premise, un pied cloud) explosent en ce moment.
🐳
Conteneurisation
Docker permet de packager une application avec tout ce dont elle a besoin — ça démarre en quelques secondes. Kubernetes gère plusieurs conteneurs en production (déploiement, redémarrage auto, load balancing). C'est la prochaine étape logique après la virtualisation, et je compte m'y mettre prochainement.
🔮
Tendances actuelles
L'infra as code (Terraform, Ansible) c'est ce qui change vraiment le métier — plus besoin de tout configurer à la main. Le Zero Trust s'impose dans les environnements hybrides. Et le Edge Computing commence à pointer son nez dans des projets IoT et industriels.

Synthèse & Impact

📈
Impact sur les métiers SISR
La virtualisation et le cloud ont transformé le rôle de l'administrateur systèmes : les compétences en déploiement automatisé, gestion de clusters et orchestration de conteneurs sont devenues indispensables sur le marché du travail.
VMware OpenStack Kubernetes
🎯
Application dans mes projets
Mes projets de virtualisation (VMware ESXi, OpenStack, KVM) m'ont permis d'appliquer concrètement les concepts étudiés. La migration live KVM illustre parfaitement les enjeux de disponibilité des services en environnement virtualisé.
KVM Live Migration vMotion HA / FT
🚀
Perspectives & Évolutions
Les prochaines étapes de ma montée en compétences incluent Docker et Kubernetes, Terraform pour l'infrastructure as code, et la certification AWS Cloud Practitioner pour maîtriser les environnements cloud publics.
Docker Kubernetes Terraform AWS
